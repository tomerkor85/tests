Platform Overview

RadixInsight is an internal event-based analytics platform designed for organizational use. It provides deep analysis of user behavior, retention, segmentation, funnels, and visual reporting.

This platform offers a complete solution for tracking and analyzing user interactions across your applications, providing valuable insights to drive business decisions.

User Segmentation

Measure and improve user retention

Data Flow & Event Tracking

The data flow architecture defines how events are collected, processed, stored, and analyzed within RadixInsight.

Event Flow Process

1

Event Generation

User interacts with application, triggering event capture by the JavaScript SDK.

2

Event Batching

SDK batches events (configurable, default: 10 events or 5 seconds).

3

API Submission

Batched events sent to API Gateway for processing.

4

Validation & Enrichment

Events validated against schema and enriched with metadata.

5

Queueing

Valid events published to Kafka/Redis Streams message queue.

6

Consumption

Events consumed from queue and prepared for storage.

7

Storage

Events written to ClickHouse database for analysis.

8

Analysis

Data queried and analyzed through the Analytics Engine.

Infrastructure Setup

RadixInsight is deployed on Google Cloud Platform (GCP) with a multi-environment architecture.

Development

  • Feature development and initial testing
  • Synthetic test data only
  • Minimal resources, cost-optimized
  • Development team access only

Staging

  • Integration testing and UAT
  • Anonymized subset of production data
  • Medium resources, scaled-down production
  • Development team and stakeholder access

Production

  • Live system for organizational use
  • Real organizational data with security controls
  • Full resources based on organizational needs
  • Restricted operations team access

Security & Privacy

RadixInsight implements comprehensive security and privacy measures to protect data and ensure compliance.

Authentication & Authorization

  • OpenID Connect (OIDC) integration
  • Multi-factor authentication (MFA)
  • Role-based access control (RBAC)
  • API key management for services

Data Protection

  • TLS 1.3 for all communications
  • Encryption at rest for all storage
  • Field-level encryption for sensitive data
  • Data anonymization techniques

Infrastructure Security

  • Private VPC with limited access
  • Container security policies
  • Secret management with rotation
  • DDoS protection with Cloud Armor

Compliance Framework

  • GDPR compliance support
  • Comprehensive audit logging
  • Vulnerability management
  • Privacy by design principles
