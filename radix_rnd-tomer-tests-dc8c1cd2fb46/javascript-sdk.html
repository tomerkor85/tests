JavaScript SDK

Lightweight client-side library for tracking events with queue management and session tracking

Overview

The RadixInsight JavaScript SDK is a lightweight client-side library that enables seamless event tracking in web applications. It provides developers with simple APIs to capture user interactions and send them to the RadixInsight platform for analysis.

Key features of the JavaScript SDK include:

  • Automatic session management and user identification
  • Event batching and queue management for optimal network usage
  • Offline support with local storage persistence
  • Automatic retry mechanism for failed network requests
  • Customizable event properties and metadata
  • Minimal footprint (less than 10KB gzipped)

Architecture Diagram

The JavaScript SDK architecture consists of several key components:

  • Event Collector: Captures and validates user interactions
  • Event Queue: Manages batching and persistence of events
  • Session Manager: Handles user session tracking and identification
  • Network Layer: Manages communication with the RadixInsight API
  • Storage Manager: Provides persistence for offline operation
  • Configuration Manager: Handles SDK initialization and settings

Input/Output Specifications

Inputs

  • Configuration: API key, endpoint URL, batch size, etc.
  • Event Data: Event name, properties, and timestamp
  • User Identification: User ID or anonymous ID
  • Context Information: Browser, device, and network details

Outputs

  • API Requests: Batched events sent to the RadixInsight API
  • Local Storage: Persisted events for offline operation
  • Debug Information: Logging and error reporting

Code Examples

Basic SDK Integration

<!-- Include the RadixInsight SDK -->
<script src="https://cdn.radixinsight.company.com/sdk/v1/radixinsight.min.js"></script>

<script>
  // Initialize the SDK
  RadixInsight.init({
    apiKey: 'YOUR_API_KEY',
    endpoint: 'https://api.radixinsight.company.com/collect',
    batchSize: 10,
    batchInterval: 5000 // milliseconds
  });
  
  // Identify the user (if known)
  RadixInsight.identify('user-123', {
    email: 'user@example.com',
    name: 'John Doe',
    plan: 'premium'
  });
  
  // Track an event
  RadixInsight.track('Button Clicked', {
    buttonId: 'signup-button',
    page: '/landing',
    referrer: document.referrer
  });
</script>

Advanced Usage

// Configure SDK with advanced options
RadixInsight.init({
  apiKey: 'YOUR_API_KEY',
  endpoint: 'https://api.radixinsight.company.com/collect',
  batchSize: 20,
  batchInterval: 10000,
  maxRetries: 3,
  retryInterval: 3000,
  debug: true,
  persistenceMode: 'localStorage',
  maxQueueSize: 100,
  autoTrack: {
    pageViews: true,
    clicks: true,
    forms: true
  },
  context: {
    app: {
      name: 'My Web App',
      version: '1.2.3'
    }
  }
});

// Track page view with custom properties
RadixInsight.pageView({
  title: document.title,
  url: window.location.href,
  referrer: document.referrer,
  loadTime: performance.now()
});

// Track form submission
const form = document.getElementById('signup-form');
form.addEventListener('submit', function(event) {
  // Prevent default form submission
  event.preventDefault();
  
  // Get form data
  const formData = new FormData(form);
  const formValues = {};
  
  for (let [key, value] of formData.entries()) {
    formValues[key] = value;
  }
  
  // Track the event
  RadixInsight.track('Form Submitted', {
    formId: form.id,
    formValues: formValues
  });
  
  // Continue with form submission
  setTimeout(() => {
    form.submit();
  }, 300);
});

API Integration

The JavaScript SDK communicates with the RadixInsight backend through the following API endpoints:

  • POST /collect - Send batched events to the platform
  • POST /identify - Update user profile information
  • GET /config - Retrieve dynamic configuration

For complete API documentation, visit the RadixInsight API Reference.

Component Relationships

Dependencies

The JavaScript SDK depends on the following components:

  • API Gateway - Receives and validates event data
  • Event Ingestion Service - Processes incoming events

Consumers

The following components consume or interact with data from the JavaScript SDK:

  • Event Consumer Service - Processes events for storage
  • Analytics Engine - Analyzes event data
  • Analytics Dashboard - Visualizes event data
